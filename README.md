# WikiClaim

Own Wikipedia articles and earn from their real-world traffic. Both parts of
this repo use the public
[Wikimedia Pageviews API](https://wikimedia.org/api/rest_v1/#/Pageviews%20data)
(human traffic only, bots excluded).

This repo contains two things:

| Part | What it is |
| ---- | ---------- |
| [**`web/`**](web) | A **multiplayer web game** — buy Wikipedia articles priced from a year of their average daily views plus a recent-traffic bonus, earn credits every day for every visitor they get, sell, and climb a shared net-worth leaderboard. |
| root (extension) | The original **browser extension** — "claim" articles you visit and rack up points from their daily views. |

---

## The web game (`web/`)

Buy articles at the price of a year of their traffic, collect their views as daily
income, and compete on net worth. Everyone starts with **5,000 credits**.

- **Price** of a page = its average daily pageviews over the last 365 days
  (min 1), plus a premium equal to its raw view total over the last 30 days
  — like valuing a business at its normal yearly run-rate plus a bonus for a
  strong recent month.
- **Buying** deducts the current price. Ownership is exclusive — once
  someone owns a page, it's off the primary market; the only way to get it
  is to buy it from them on the **secondary market**, at whatever price
  they choose to list it for.
- **Earning**: each real calendar day, every page you own pays you credits equal
  to that day's view count. Settlement is automatic and lazy — loading your
  dashboard credits every page for each day published since it was last settled,
  so a page owned N days earns exactly N days of real traffic. Wikimedia lags
  ~1 day, so today's views pay out tomorrow.
- **Selling** returns the page's current price instantly; prices drift with
  real traffic. Listing on the secondary market instead is a direct,
  peer-to-peer trade — the buyer's payment goes straight to you, not "the
  market" — so it can sell for more or less than the instant-sell price.
- **Net worth** = credits + current value of your pages; the leaderboard ranks
  everyone by it.

### Run it

Requires Node.js 18+ (uses built-in `fetch`).

```bash
cd web
npm install
npm start          # http://localhost:3000
```

With no `DATABASE_URL` it uses a local JSON file (zero setup); set
`DATABASE_URL` to use Postgres. Deployable to Railway with a Postgres plugin —
see [`web/README.md`](web/README.md) for the storage backends, Railway steps,
API reference, and architecture.

## The browser extension (repo root)

A Manifest V3 extension with two independent features:

- **The corner widget** (`content.js`) reads live data from the WikiPicks web
  game (`GET /api/article` on wikipicks.app) for whatever article you're on:
  who owns it (and its resale price, if listed), or its claim price and
  yesterday's views if it's unclaimed. Since ownership is exclusive and
  claiming spends real WikiPicks credits, the claim/buy button doesn't
  transact from the widget itself — it opens the article on wikipicks.app,
  where you sign in (if needed) and confirm the purchase through the game's
  own flow. Only English Wikipedia is tracked; other language editions show
  a short note instead.
- **The toolbar popup** (`popup.html`/`.js`) is a separate, older, purely
  local point tracker — unrelated to WikiPicks accounts or ownership. Points
  are stored per-browser in `chrome.storage.local`, synced from the public
  Pageviews API on a 4h alarm.

### Install (unpacked)

1. Open `chrome://extensions` (or `edge://extensions`).
2. Enable **Developer mode**.
3. Click **Load unpacked** and select this repo's root folder.
4. Visit e.g. `https://en.wikipedia.org/wiki/Cat` — the corner widget shows
   its WikiPicks status and lets you jump there to claim it.
5. Click the toolbar icon for the local point dashboard; the ⟳ button forces
   a sync.

### Layout

- `manifest.json` — MV3 manifest.
- `background.js` — service worker: local claim storage + pageview sync (4h
  alarm + startup) for the popup, plus a proxy fetch to wikipicks.app's
  public `/api/article` for the on-page widget. Message handling for both.
- `content.js` / `content.css` — the on-page widget showing live WikiPicks
  ownership/price/views (see above).
- `popup.html` / `popup.css` / `popup.js` — the toolbar's local-points dashboard.
- `lib/wiki.js` — shared URL-parsing + Pageviews API helpers.
- `icons/` — generated by `scripts/gen_icons.py` (dependency-free PNG writer).

Notes: only plain article pages count (`Special:`, `Talk:`, etc. are ignored).
